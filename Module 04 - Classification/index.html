Module 4, Classification

When the response is a category, not a number, and a straight line is no longer the right tool.

Module 3 assumed a quantitative response, sales, yield, spend. Many of the most important problems in practice instead have a qualitative response, a category, fraud or not fraud, default or no default, which of three possible causes explains a failed transaction. This module builds the main tools for that setting.

By the end of this module you should be able to

  • Explain why ordinary linear regression is the wrong tool for a categorical response
  • Fit and interpret a logistic regression model, including the logit transformation and maximum likelihood estimation
  • Extend logistic regression to several predictors, multiple classes, and recognize confounding between predictors
  • Derive and apply linear and quadratic discriminant analysis, and the Naive Bayes classifier
  • Build a confusion matrix, compute error rates, and read an ROC curve to judge a classifier honestly
  • Compare logistic regression, LDA, QDA, and Naive Bayes, and know when each is the more sensible choice
A running example for this module: MobiPay, a fictional mobile financial services app, wants to flag fraudulent transactions automatically, using each transaction's amount and the customer's account age as predictors.

The Classification Problem

In classification, the response \(Y\) is qualitative, it takes values in an unordered, finite set of categories \(C\), rather than a continuous numeric scale. Examples include a customer's loan outcome, default or no default, a transaction's status, fraudulent or legitimate, or a crop's disease status, healthy, blighted, or pest damaged.

Given a feature vector \(X\), the classification task is to build a function \(C(X)\) that predicts \(Y\). Often we care about more than just a single hard prediction, we want the estimated probability that \(Y\) belongs to each class, since that probability carries information a single label throws away.

Why probabilities matter, not just labels: telling a risk team "this transaction has an 8 percent chance of being fraudulent" is far more useful than a flat "not fraud", since it lets them rank transactions by risk and set a sensible review threshold.

Why Not Just Use Linear Regression?

A tempting shortcut is to code the response numerically, say \(Y = 1\) for fraud and \(Y = 0\) for legitimate, and fit ordinary linear regression of \(Y\) on the predictors, then classify as fraud whenever the predicted \(\hat Y > 0.5\).

For a strictly two-class response, this turns out to be closely related to linear discriminant analysis, covered later in this module, and can sometimes work reasonably. But it has a basic flaw: nothing forces the predicted value to stay between 0 and 1.

The core problem: a linear regression fit to a 0/1 response can predict negative probabilities for very small transaction amounts, or probabilities above 1 for very large ones, numbers that are not valid probabilities at all.

Linear Regression Fails Even More for Multiple Classes

Suppose instead MobiPay wants to classify the likely cause of a disputed transaction into three unordered categories, network timeout, agent error, or fraud attempt. A tempting numeric coding might be:

CodeCause
1Network timeout
2Agent error
3Fraud attempt

This coding silently implies an ordering, fraud attempt is "more" than agent error, which is "more" than network timeout, and it implies the gap between codes 1 and 2 is the same size as the gap between codes 2 and 3. Neither implication has any real meaning here, the three causes are simply different, not ordered or equally spaced.

The takeaway: for an unordered categorical response with more than two levels, linear regression is not just imperfect, it is the wrong kind of model entirely. A different coding, say swapping which cause gets code 1 versus code 3, would produce a completely different fitted model for the same data.

The Logistic Function

Logistic regression instead models the probability directly, using a function guaranteed to stay between 0 and 1 for any input. For a single predictor \(X\):

\[ p(X) = \frac{e^{\beta_0 + \beta_1 X}}{1 + e^{\beta_0 + \beta_1 X}} \]

Here \(e \approx 2.71828\) is Euler's number. No matter how large or small \(\beta_0 + \beta_1 X\) becomes, the expression \(\frac{e^{z}}{1+e^{z}}\) always lands strictly between 0 and 1, exactly what a probability needs.

Shape of the curve: as the predictor grows very large, \(p(X)\) approaches 1, as it grows very negative, \(p(X)\) approaches 0, and the curve is a smooth S-shape, steepest near the point where \(p(X) = 0.5\).

The Log-Odds, or Logit, Transformation

Rearranging the logistic function gives an equivalent, very useful form. Define the odds of the event as \(\frac{p(X)}{1-p(X)}\), then:

\[ \log\left(\frac{p(X)}{1-p(X)}\right) = \beta_0 + \beta_1 X \]

The left side is called the log-odds or logit of \(p(X)\). This transformation is the key trick behind logistic regression, it converts a probability, which is awkwardly bounded between 0 and 1, into a quantity that behaves like an ordinary linear regression response, free to range over all real numbers.

Reading the coefficient: \(\beta_1\) is the change in log-odds of the event for a one unit increase in \(X\), not directly a change in probability. This is exactly why interpreting logistic regression coefficients takes a little more care than linear regression coefficients.

Fitting Logistic Regression, Maximum Likelihood

Unlike least squares, logistic regression coefficients are fit using maximum likelihood estimation. The likelihood function describes how probable the observed pattern of zeros and ones would be, for a given choice of \(\beta_0, \beta_1\):

\[ \ell(\beta_0, \beta_1) = \prod_{i: y_i = 1} p(x_i) \prod_{i: y_i = 0} (1 - p(x_i)) \]

We choose the coefficients \(\hat\beta_0, \hat\beta_1\) that make this likelihood as large as possible, the values under which the data we actually observed would have been the most probable outcome. Most statistical software fits this automatically, for example with R's glm function.

Making Predictions, a Worked Example

Suppose fitting logistic regression of fraud status on transaction amount (in taka) for MobiPay gives \(\hat\beta_0 = -6.5\) and \(\hat\beta_1 = 0.00018\). For a transaction of 10,000 taka:

\[ \hat p(X) = \frac{e^{-6.5 + 0.00018 \times 10000}}{1 + e^{-6.5 + 0.00018 \times 10000}} = \frac{e^{-4.7}}{1+e^{-4.7}} \approx 0.009 \]

For a much larger transaction of 40,000 taka, the estimated probability of fraud rises sharply:

\[ \hat p(X) = \frac{e^{-6.5 + 0.00018 \times 40000}}{1 + e^{-6.5 + 0.00018 \times 40000}} = \frac{e^{0.7}}{1+e^{0.7}} \approx 0.668 \]

A small change in transaction amount can shift the estimated probability dramatically, this is the steep, S-shaped middle of the logistic curve at work.

Multiple Logistic Regression

Just as with linear regression, logistic regression extends naturally to several predictors \(X_1, \ldots, X_p\):

\[ \log\left(\frac{p(X)}{1-p(X)}\right) = \beta_0 + \beta_1 X_1 + \cdots + \beta_p X_p, \qquad p(X) = \frac{e^{\beta_0+\beta_1X_1+\cdots+\beta_pX_p}}{1+e^{\beta_0+\beta_1X_1+\cdots+\beta_pX_p}} \]

Suppose MobiPay adds a second predictor, whether the account is an authorized merchant agent account, to the transaction amount model. The fitted coefficients might look like this:

CoefficientStd. Errorz-statisticp-value
Intercept-6.900.31-22.26< 0.0001
amount0.000190.00000823.75< 0.0001
agent[Yes]-0.810.24-3.380.0007

Each coefficient is now interpreted holding the other predictors fixed, exactly as in multiple linear regression. The negative agent coefficient says that, for a fixed transaction amount, agent accounts are associated with a lower estimated fraud probability than non-agent accounts.

Confounding, a Sign Can Flip

Now suppose a simpler model uses only the agent status by itself, ignoring transaction amount, and the fitted agent coefficient comes out positive, the opposite sign from the multiple predictor model on the previous slide. Both fits can be correct at once. Why?

Confounding explained: agent accounts tend to process much larger transactions than ordinary customer accounts, on average, and larger transactions are independently riskier. So agent accounts look riskier overall (the marginal relationship), even though, for a fixed transaction amount, agent accounts actually default less often than non-agent accounts (the partial relationship).

This is the same confounding idea seen in Module 3, where a predictor's marginal relationship with the response can look completely different from its partial relationship once a correlated predictor is added to the model.

Logistic Regression With More Than Two Classes

Logistic regression generalizes to \(K\) classes. One common symmetric form, used for example in the glmnet package, fits a linear function for each class:

\[ \Pr(Y=k \mid X) = \frac{e^{\beta_{0k}+\beta_{1k}X_1+\cdots+\beta_{pk}X_p}}{\sum_{\ell=1}^{K} e^{\beta_{0\ell}+\beta_{1\ell}X_1+\cdots+\beta_{p\ell}X_p}} \]

This is also called multinomial logistic regression. Applied to MobiPay's disputed transaction example, network timeout, agent error, or fraud attempt, the model would estimate three probabilities for every disputed transaction, one per cause, that together sum to 1.

A technical note: there is some redundancy in this symmetric form, only \(K-1\) linear functions are actually needed, exactly as ordinary two-class logistic regression only needs one.

A Different Strategy, Discriminant Analysis

Logistic regression models \(\Pr(Y \mid X)\) directly. Discriminant analysis takes a different route: model the distribution of the predictors \(X\) within each class separately, then use Bayes theorem to flip the relationship around and recover \(\Pr(Y \mid X)\).

When we assume the predictors follow a normal (Gaussian) distribution within each class, this leads to linear discriminant analysis (LDA) or, with a small change, quadratic discriminant analysis (QDA). The general idea is not limited to Gaussian distributions, but that is our focus here.

Bayes Theorem for Classification

Bayes theorem, named for the mathematician Thomas Bayes, gives a way to reverse a conditional probability:

\[ \Pr(Y=k \mid X=x) = \frac{\Pr(X=x \mid Y=k)\cdot \Pr(Y=k)}{\Pr(X=x)} \]

For discriminant analysis this is usually written as:

\[ \Pr(Y=k \mid X=x) = \frac{\pi_k f_k(x)}{\sum_{\ell=1}^{K} \pi_\ell f_\ell(x)} \]

Here \(f_k(x) = \Pr(X=x \mid Y=k)\) is the density of \(X\) within class \(k\), and \(\pi_k = \Pr(Y=k)\) is the marginal, or prior, probability of class \(k\) before seeing any predictor at all.

Classify to the Highest Density

Given the class densities and priors, a new observation is classified to whichever class has the highest value of \(\pi_k f_k(x)\). When the prior probabilities \(\pi_k\) differ across classes, the decision boundary shifts toward the class with the larger prior, since that class needs less evidence from \(X\) to be favored.

Intuition: if MobiPay knows fraud is rare overall, say only 1 in 2000 transactions, that low prior probability of fraud should pull the decision boundary so that only quite strong evidence in \(X\) is enough to flag a transaction, rather than treating fraud and legitimate transactions as equally likely before seeing any data.

Why Discriminant Analysis?

  • When the classes are well separated, logistic regression's parameter estimates become surprisingly unstable. LDA does not suffer from this problem in the same way.
  • If the sample size \(n\) is small and the predictors are approximately normal within each class, LDA tends to be more stable than logistic regression.
  • LDA is popular when there are more than two response classes, since it also provides a natural low dimensional view of the data for visualization.
So which should you reach for? Neither method dominates the other in general, the choice depends on sample size, how separated the classes are, and how reasonable the normality assumption is, a theme this module returns to near the end.

Linear Discriminant Analysis When p = 1

With a single predictor \(X\), assume the Gaussian density within class \(k\) has mean \(\mu_k\) and variance \(\sigma_k^2\), and further assume all classes share a common variance \(\sigma^2\):

\[ f_k(x) = \frac{1}{\sqrt{2\pi}\,\sigma} \exp\left(-\frac{1}{2}\left(\frac{x-\mu_k}{\sigma}\right)^2\right) \]

Plugging this into Bayes formula gives a complicated looking expression for \(p_k(x) = \Pr(Y=k \mid X=x)\), but a lot of it cancels out cleanly, leading to a surprisingly simple decision rule, covered next.

The Discriminant Function

After taking logs and discarding terms that do not depend on \(k\), classifying to the class with the largest \(p_k(x)\) is equivalent to classifying to the class with the largest discriminant score:

\[ \delta_k(x) = x\cdot\frac{\mu_k}{\sigma^2} - \frac{\mu_k^2}{2\sigma^2} + \log(\pi_k) \]

Notice that \(\delta_k(x)\) is a linear function of \(x\), this is exactly why the method is called linear discriminant analysis. With two classes and equal priors \(\pi_1 = \pi_2\), the decision boundary works out to be the midpoint \(x = \frac{\mu_1+\mu_2}{2}\), the natural cutoff exactly halfway between the two class means.

Estimating the Parameters

In practice we never know \(\pi_k\), \(\mu_k\), and \(\sigma^2\) exactly, we estimate them from training data with \(n\) total observations and \(n_k\) observations in class \(k\):

\[ \hat\pi_k = \frac{n_k}{n}, \qquad \hat\mu_k = \frac{1}{n_k}\sum_{i:y_i=k} x_i, \qquad \hat\sigma^2 = \frac{1}{n-K}\sum_{k=1}^{K}\sum_{i:y_i=k}(x_i-\hat\mu_k)^2 \]

These plug-in estimates, the class proportions, class-specific means, and a pooled estimate of the shared variance, are then substituted into the discriminant function from the previous slide to produce a working classifier.

Linear Discriminant Analysis When p > 1

With several predictors, the Gaussian density and discriminant function generalize using vectors and matrices, the mean becomes a vector \(\mu_k\), and all classes share one covariance matrix \(\Sigma\):

\[ f(x) = \frac{1}{(2\pi)^{p/2}|\Sigma|^{1/2}} \exp\left(-\frac{1}{2}(x-\mu)^T \Sigma^{-1}(x-\mu)\right) \] \[ \delta_k(x) = x^T \Sigma^{-1}\mu_k - \frac{1}{2}\mu_k^T \Sigma^{-1}\mu_k + \log \pi_k \]

Despite the matrix notation, \(\delta_k(x)\) is still, in the end, just a linear function of the predictors, \(\delta_k(x) = c_{k0} + c_{k1}x_1 + c_{k2}x_2 + \cdots + c_{kp}x_p\), which is why the decision boundaries LDA produces are always straight lines (or flat planes, in higher dimensions).

An Illustration, Classifying Rice Variety by Grain Shape

Agricultural researchers often classify rice variety from grain measurements alone. Suppose grain length and grain width are measured for samples of three BRRI rice varieties, with roughly 50 samples per variety, and the goal is to classify a new grain sample by variety using only its measurements.

With two predictors and three classes, LDA fits three linear discriminant functions and classifies each new grain sample to whichever variety's discriminant score is highest, the dashed lines separating the three regions in such a plot are the estimated decision boundaries.

From Discriminant Scores to Probabilities

Once we have estimated discriminant scores \(\hat\delta_k(x)\) for every class, they convert into estimated class probabilities through a softmax-style transformation:

\[ \widehat{\Pr}(Y=k \mid X=x) = \frac{e^{\hat\delta_k(x)}}{\sum_{\ell=1}^{K} e^{\hat\delta_\ell(x)}} \]

Classifying to the largest \(\hat\delta_k(x)\) is exactly the same as classifying to the class with the largest estimated probability. With \(K=2\) classes, this reduces to the familiar rule, classify to class 2 if \(\widehat{\Pr}(Y=2 \mid X=x) \geq 0.5\), otherwise classify to class 1.

Reading a Confusion Matrix

Suppose LDA is fit to classify 10,000 MobiPay transactions as fraud or not fraud, using transaction amount and account age. Comparing predictions against the later-confirmed truth gives a confusion matrix:

Predicted \ TrueNo fraudFraudTotal
No fraud9,644819,725
Fraud23252275
Total9,66733310,000

The overall misclassification rate is \((23+81)/10000 = 1.04\) percent. But that single number hides something important, look closely at how the two kinds of mistakes are split.

Two Very Different Kinds of Error

False positive rate: the fraction of legitimate transactions wrongly flagged as fraud, here \(23/9667 \approx 0.24\) percent. False negative rate: the fraction of actual fraud cases the model misses entirely, here \(81/333 \approx 24.3\) percent.

Why the overall error rate is misleading: a model that always predicts "no fraud" would only make 333 errors out of 10,000, a 3.33 percent overall error rate, deceptively close to looking reasonable, yet it would catch zero actual fraud cases. For a rare event like fraud, the false negative rate often matters far more than the overall error rate.

Varying the Threshold

The confusion matrix above used the default rule, classify as fraud if \(\widehat{\Pr}(\text{fraud} \mid X) \geq 0.5\). Nothing forces us to use 0.5. Lowering the threshold, say to 0.1, flags more transactions as fraud, catching more true fraud cases (a lower false negative rate) but also wrongly flagging more legitimate transactions (a higher false positive rate).

A practical decision for MobiPay's risk team: since missing real fraud is usually far costlier than briefly delaying a few legitimate transactions for review, MobiPay may deliberately choose a lower threshold than 0.5, accepting more false alarms in exchange for catching more real fraud.

The ROC Curve and AUC

An ROC curve (receiver operating characteristic curve) plots the true positive rate against the false positive rate as the threshold varies continuously from 0 to 1, summarizing every possible tradeoff in a single picture, rather than committing to one threshold in advance.

The AUC, area under the ROC curve, summarizes overall classifier quality in a single number between 0 and 1, a perfect classifier has AUC = 1, a classifier no better than random guessing has AUC = 0.5. Higher AUC means better separation between the two classes across all thresholds at once.

Quadratic Discriminant Analysis (QDA)

LDA assumes every class shares the same covariance matrix \(\Sigma\). QDA relaxes this, allowing each class its own covariance matrix \(\Sigma_k\):

\[ \delta_k(x) = -\frac{1}{2}(x-\mu_k)^T \Sigma_k^{-1}(x-\mu_k) + \log \pi_k \]

Because each \(\Sigma_k\) can differ, the quadratic terms no longer cancel out, so \(\delta_k(x)\) is now a genuinely quadratic function of \(x\), giving QDA curved decision boundaries instead of LDA's straight lines. This added flexibility helps when classes truly have different spreads, but it costs more parameters to estimate, so QDA needs more training data to do so reliably than LDA does.

Naive Bayes

When the number of predictors \(p\) is large, even LDA and QDA can struggle, since both require estimating a covariance structure between every pair of predictors. Naive Bayes sidesteps this by assuming the predictors are independent within each class:

\[ f_k(x) = \prod_{j=1}^{p} f_{kj}(x_j) \]

For Gaussian features this means treating each \(\Sigma_k\) as if it were diagonal, no covariance terms between predictors at all, only individual variances. This assumption is rarely exactly true, transaction amount and time of day are not really independent, but despite the strong simplification, Naive Bayes often classifies surprisingly well in practice, and it scales easily to a large number of predictors and even to mixed qualitative and quantitative features.

Logistic Regression Versus LDA

For a two-class problem, LDA's log-odds also turns out to be a linear function of \(x\):

\[ \log\left(\frac{p_1(x)}{1-p_1(x)}\right) = c_0 + c_1 x_1 + \cdots + c_p x_p \]

This is exactly the same form as logistic regression. The real difference is in how the parameters are estimated. Logistic regression maximizes the conditional likelihood based on \(\Pr(Y \mid X)\), known as discriminative learning. LDA instead maximizes the full likelihood based on \(\Pr(X, Y)\), known as generative learning, modeling how the predictors themselves are distributed.

In practice: despite this conceptual difference, the two methods often give very similar results. A footnote worth remembering: logistic regression can also fit curved, QDA-like boundaries, simply by explicitly adding quadratic terms to the model.

Comparing the Classifiers, a Quick Reference

Logistic Regression

  • Very popular default choice, especially with two classes
  • Makes no assumption about the distribution of the predictors
  • Becomes unstable when classes are very well separated

LDA

  • More stable with small \(n\) or well-separated classes
  • Convenient with more than two classes
  • Assumes shared covariance and approximate normality

QDA

  • More flexible than LDA, curved decision boundaries
  • Needs more data to estimate reliably
  • Useful when classes clearly have different spreads

Naive Bayes

  • Scales well when \(p\) is very large
  • Assumes predictors are independent within each class
  • Often performs well despite the strong assumption

What These Methods Cannot Do Well

  • All the methods in this module assume a fairly simple decision boundary shape, linear for logistic regression and LDA, quadratic for QDA, none of them can easily capture an arbitrarily curved boundary
  • LDA and QDA rely on a normality assumption for the predictors that can fail badly for skewed or heavy-tailed features
  • Naive Bayes can perform poorly when predictors are strongly dependent within a class, despite often working well in practice
  • None of these methods automatically tell you which predictors actually matter when there are very many candidates
Looking ahead: later modules introduce resampling methods to honestly evaluate how well any of today's classifiers will generalize beyond the training data, variable selection and regularization for when there are many candidate predictors, and tree-based and other flexible methods that can capture decision boundaries no straight line or simple curve ever could.
