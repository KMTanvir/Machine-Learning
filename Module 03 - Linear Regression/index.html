Module 3, Linear Regression

The first concrete method for estimating f, and the method every later module will be compared against.

Module 2 gave you the tools to judge any model, mean squared error, the bias variance decomposition, and the probability foundations underneath them. Linear regression is where those tools get used for the first time, on the simplest model that can actually estimate a relationship between inputs and an output.

By the end of this module you should be able to

  • Fit and interpret a simple linear regression, and explain what least squares is minimizing
  • Compute and interpret standard errors, confidence intervals, and hypothesis tests for regression coefficients
  • Assess overall model fit using RSE and R-squared
  • Extend to multiple linear regression, and interpret coefficients correctly when predictors are correlated
  • Use qualitative predictors, interaction terms, and polynomial terms to extend the basic linear model
  • Recognize the main things that can go wrong with a fitted regression, non-linearity, outliers, high leverage points, and collinearity
A famous quote worth keeping in mind: "Essentially, all models are wrong, but some are useful," George Box. Linear regression is almost never exactly true, but it is often extremely useful.

Simple Linear Regression, the Model

Simple linear regression uses a single predictor \(X\) to estimate a quantitative response \(Y\), assuming the relationship is approximately a straight line.

\[ Y = \beta_0 + \beta_1 X + \varepsilon \]

Here \(\beta_0\) and \(\beta_1\) are unknown constants, the intercept and the slope, also called coefficients or parameters, and \(\varepsilon\) is the same kind of error term from Module 2, independent of \(X\), with mean zero.

An honest caveat, stated upfront: true relationships in nature are essentially never exactly linear. Linear regression is best understood as a useful, interpretable approximation, not a claim that the world is actually built from straight lines.

A Running Example, Advertising and Sales

Throughout this module we will use a single running example, a company's advertising budget, spent across TV, radio, and newspaper, used to predict product sales. The questions this dataset lets us ask are exactly the questions linear regression is built to answer.

  • Is there a relationship between advertising spend and sales at all?
  • How strong is that relationship, and is it the same for every channel?
  • Which channels actually matter, once the others are accounted for?
  • How accurately can we predict future sales from a proposed budget?
  • Is the relationship truly linear, and do the channels interact?
Why a single running example helps: every formula in this module will be applied to the exact same numbers, so you can see how each new idea changes, or confirms, the previous conclusion.

Least Squares Estimation

Given estimates \(\hat{\beta}_0\) and \(\hat{\beta}_1\), we predict sales using \(\hat{y} = \hat{\beta}_0 + \hat{\beta}_1 x\). The difference between an actual value and its prediction is the residual, \(e_i = y_i - \hat{y}_i\).

The residual sum of squares, RSS, adds up the squared residuals across all \(n\) observations,

\[ RSS = e_1^2 + e_2^2 + \cdots + e_n^2 \]

The least squares approach chooses \(\hat{\beta}_0\) and \(\hat{\beta}_1\) to make RSS as small as possible. This is exactly the same minimization idea as the error function \(E(w)\) from polynomial curve fitting in Module 2, just renamed and specialized to a straight line.

The minimizing values have a closed form,

\[ \hat{\beta}_1 = \frac{\sum_{i=1}^{n}(x_i - \bar{x})(y_i - \bar{y})}{\sum_{i=1}^{n}(x_i - \bar{x})^2}, \qquad \hat{\beta}_0 = \bar{y} - \hat{\beta}_1 \bar{x} \]

A Worked Example, by Hand

Suppose we record weekly mobile data top up spend (in hundreds of taka) and weekly data usage (in GB) for five customers.

Here \(\bar{x} = 6\), \(\bar{y} = 7\). Computing the needed sums, \(\sum (x_i-\bar{x})(y_i-\bar{y}) = (-4)(-4)+(-2)(-2)+0+2\cdot1+4\cdot5 = 16+4+0+2+20 = 42\), and \(\sum(x_i-\bar{x})^2 = 16+4+0+4+16 = 40\).

\[ \hat{\beta}_1 = \frac{42}{40} = 1.05, \qquad \hat{\beta}_0 = 7 - 1.05 \times 6 = 0.7 \]

So the fitted line is \(\hat{y} = 0.7 + 1.05x\). Each additional hundred taka of top up spend is associated with about 1.05 GB more usage, on average.

Assessing the Accuracy of the Coefficient Estimates

\(\hat{\beta}_0\) and \(\hat{\beta}_1\) are estimates, computed from one particular sample, so they carry uncertainty. The standard error of an estimator describes how much it would vary if we repeated the sampling process.

\[ SE(\hat{\beta}_1)^2 = \frac{\sigma^2}{\sum_{i=1}^{n}(x_i-\bar{x})^2}, \qquad SE(\hat{\beta}_0)^2 = \sigma^2\left[\frac{1}{n} + \frac{\bar{x}^2}{\sum_{i=1}^{n}(x_i-\bar{x})^2}\right] \]

where \(\sigma^2 = \text{Var}(\varepsilon)\), the irreducible error variance from Module 2, now appearing in a very concrete, practical formula.

Intuition: \(SE(\hat{\beta}_1)\) shrinks when the \(x_i\) are more spread out, since a wider spread of \(x\) values pins down the slope more precisely.

Confidence Intervals for Coefficients

Standard errors let us build a confidence interval, a range of plausible values for the true, unknown coefficient. An approximate 95 percent confidence interval for \(\beta_1\) is

\[ \hat{\beta}_1 \pm 2 \cdot SE(\hat{\beta}_1) \]

This means, if we could repeat the data collection many times and build this interval each time, about 95 percent of those intervals would contain the true \(\beta_1\). For the advertising data, fitting sales on TV budget gives a 95 percent confidence interval for \(\beta_1\) of approximately \([0.042,\, 0.053]\).

A common misreading to avoid: the confidence interval is a statement about the procedure across repeated samples, not a 95 percent probability statement about this one fixed, already computed interval containing the truth.

Hypothesis Testing on Coefficients

The most common test asks whether there is any relationship at all between \(X\) and \(Y\),

\[ H_0: \beta_1 = 0 \qquad \text{versus} \qquad H_A: \beta_1 \ne 0 \]

If \(\beta_1 = 0\), the model collapses to \(Y = \beta_0 + \varepsilon\), and \(X\) carries no information about \(Y\). We compute a t-statistic,

\[ t = \frac{\hat{\beta}_1 - 0}{SE(\hat{\beta}_1)} \]

and compare it to a t-distribution with \(n-2\) degrees of freedom to get a p-value, the probability of seeing a \(t\) this large or larger if \(H_0\) were actually true. A small p-value, conventionally below 0.05, is evidence against \(H_0\).

Worked Example, Reading a Coefficient Table

Fitting sales on TV advertising budget alone gives the following table.

The TV coefficient's t-statistic is \(0.0475 / 0.0027 \approx 17.67\), an extremely large value, giving a p-value far below 0.0001. We reject \(H_0: \beta_1 = 0\) and conclude there is strong evidence of a relationship between TV advertising and sales.

Assessing the Overall Accuracy of the Model

Beyond individual coefficients, we want a sense of how well the model fits overall. The residual standard error, RSE, estimates the typical size of a residual, in the units of \(Y\),

\[ RSE = \sqrt{\frac{1}{n-2} RSS} \]

The R-squared statistic instead reports the proportion of variance in \(Y\) explained by the model, on a scale from 0 to 1,

\[ R^2 = \frac{TSS - RSS}{TSS} = 1 - \frac{RSS}{TSS}, \qquad TSS = \sum_{i=1}^{n}(y_i - \bar{y})^2 \]

In simple linear regression, \(R^2\) is exactly the squared correlation between \(X\) and \(Y\).

From Simple to Multiple Linear Regression

Real problems usually involve several predictors at once. Multiple linear regression extends the model directly,

\[ Y = \beta_0 + \beta_1 X_1 + \beta_2 X_2 + \cdots + \beta_p X_p + \varepsilon \]

For the advertising data with all three channels, the model becomes

\[ \text{sales} = \beta_0 + \beta_1 \times \text{TV} + \beta_2 \times \text{radio} + \beta_3 \times \text{newspaper} + \varepsilon \]

Each \(\beta_j\) is interpreted as the average effect on \(Y\) of a one unit increase in \(X_j\), holding all the other predictors fixed.

The Woes of Interpreting Regression Coefficients

The phrase "holding all other predictors fixed" sounds clean, but real predictors usually move together, which makes interpretation genuinely tricky.

Example, pocket change: let \(Y\) be the total value of coins in your pocket, \(X_1\) the number of coins, and \(X_2\) the number of low value coins. Regressing \(Y\) on \(X_2\) alone gives a positive coefficient. But once \(X_1\) is included too, the coefficient on \(X_2\) can become negative, holding the total number of coins fixed, more low value coins simply means less total value.

The lesson, attributed to Mosteller and Tukey, is that a coefficient describes a partial, holding-fixed relationship, not the simple, marginal one you would see by looking at \(X_j\) and \(Y\) alone.

A caution about causality: correlated predictors make causal claims especially risky from observational data. "The only way to find out what happens when you disturb a system is to disturb it, not merely observe it," as Mosteller and Tukey put it, paraphrasing George Box.

Estimation in Multiple Regression

Exactly as before, the coefficients are chosen to minimize the residual sum of squares,

\[ RSS = \sum_{i=1}^{n}\left(y_i - \hat{\beta}_0 - \hat{\beta}_1 x_{i1} - \hat{\beta}_2 x_{i2} - \cdots - \hat{\beta}_p x_{ip}\right)^2 \]

With more than one predictor, the formulas for \(\hat{\beta}_0, \dots, \hat{\beta}_p\) are most naturally written using matrix algebra, and are always computed by statistical software rather than by hand. What matters conceptually is that the goal is unchanged, minimize the same kind of squared error from Module 2, now over a higher dimensional fitted plane rather than a single line.

Four Important Questions in Multiple Regression

  1. Is at least one of the predictors \(X_1, \dots, X_p\) useful for predicting the response?
  2. Do all the predictors help explain \(Y\), or only a subset of them?
  3. How well does the model fit the data overall?
  4. Given a set of predictor values, what response should we predict, and how accurate is that prediction?
Why these four questions matter: they are the same four questions you should ask of any fitted model in this course, not just linear regression, only the specific tools used to answer them change from method to method.

Is at Least One Predictor Useful, the F-statistic

To test whether any of the predictors are related to the response, we test \(H_0: \beta_1 = \beta_2 = \cdots = \beta_p = 0\) using the F-statistic,

\[ F = \frac{(TSS - RSS)/p}{RSS/(n-p-1)} \]

If \(H_0\) is true and the usual assumptions hold, \(F\) is expected to be close to 1. A value of \(F\) well above 1 is evidence that at least one predictor is genuinely related to the response.

For the advertising data with all three channels, the model achieves RSE = 1.69, \(R^2 = 0.897\), and \(F = 570\), an enormous F-statistic, strong evidence that the model as a whole is informative.

Deciding on the Important Variables

With \(p\) predictors there are \(2^p\) possible subsets of predictors to consider, for \(p = 40\) that is already over a billion models, far too many to fit and compare directly. Two simple, automated strategies are commonly used instead.

Forward selection

  • Start with no predictors, just an intercept
  • Add the predictor that most improves the fit
  • Keep adding one at a time until a stopping rule is met

Backward selection

  • Start with all predictors included
  • Remove the least significant predictor, the one with the largest p-value
  • Keep removing one at a time until a stopping rule is met

More systematic criteria for comparing models along the way, such as adjusted \(R^2\) and cross validation, are covered later in the course.

Qualitative Predictors

Not every predictor is a number. Qualitative or categorical predictors take a discrete set of values, for example, whether a customer is a mobile banking agent or not, or which administrative division they belong to.

To use a two level qualitative predictor in a regression, we create a dummy variable,

\[ x_i = \begin{cases} 1 & \text{if condition A holds for observation } i \\ 0 & \text{otherwise} \end{cases} \]

The model \(y_i = \beta_0 + \beta_1 x_i + \varepsilon_i\) then reduces to two separate intercepts, \(\beta_0 + \beta_1\) when the condition holds, and \(\beta_0\) otherwise. \(\beta_1\) becomes the average difference between the two groups.

Worked Example, a Two Level Dummy Variable

Using a credit card balance dataset, define \(x_i = 1\) if the customer is female and \(0\) if male.

Qualitative Predictors with More than Two Levels

With more than two levels, we need additional dummy variables, one fewer than the number of levels. For a three level ethnicity variable, Asian, Caucasian, African American, we create two dummies,

\[ x_{i1} = \begin{cases} 1 & \text{Asian} \\ 0 & \text{otherwise} \end{cases}, \qquad x_{i2} = \begin{cases} 1 & \text{Caucasian} \\ 0 & \text{otherwise} \end{cases} \]

The level left without its own dummy, African American here, becomes the baseline, against which the other levels are compared. \(\beta_0\) is the baseline group's average, and \(\beta_1, \beta_2\) are differences from that baseline.

Removing the Additive Assumption, Interactions

The additive multiple regression model assumes the effect of one predictor on \(Y\) does not depend on the level of another predictor. For advertising, the additive model says the effect of TV on sales is always \(\beta_1\), regardless of how much is spent on radio.

But suppose radio spending actually makes TV advertising more effective, so that splitting a fixed budget between the two media boosts sales more than spending it all on either one alone. In marketing this is called a synergy effect, in statistics, an interaction effect.

How to spot it informally: if a model systematically underestimates \(Y\) exactly when two predictors are both at moderate, mixed levels, and overestimates \(Y\) when one predictor dominates, that pattern is a classic sign of a missing interaction term.

Modelling an Interaction

An interaction term is added by including the product of the two predictors,

\[ \text{sales} = \beta_0 + \beta_1 \times \text{TV} + \beta_2 \times \text{radio} + \beta_3 \times (\text{TV} \times \text{radio}) + \varepsilon \]

which can be rearranged as \(\text{sales} = \beta_0 + (\beta_1 + \beta_3 \times \text{radio}) \times \text{TV} + \beta_2 \times \text{radio} + \varepsilon\), so the effective slope on TV now depends on the level of radio.

CoefficientStd. Errort-statisticp-value
Intercept6.75020.24827.23< 0.0001
TV0.01910.00212.70< 0.0001
radio0.02890.0093.240.0014
TV × radio0.00110.00020.73< 0.0001

Interpreting the Interaction Model

The interaction term's p-value is extremely small, strong evidence of a genuine synergy effect. The interaction model's \(R^2\) is 96.8 percent, compared to 89.7 percent for the additive model with just TV and radio.

That gap means the interaction term explains \((96.8 - 89.7)/(100 - 89.7) \approx 69\) percent of the variance left over after the additive model, a substantial improvement.

With these coefficients, a $1,000 increase in TV spend is associated with an increase in sales of \((\hat{\beta}_1 + \hat{\beta}_3 \times \text{radio}) \times 1000\) units, and a $1,000 increase in radio spend is associated with an increase of \((\hat{\beta}_2 + \hat{\beta}_3 \times \text{TV}) \times 1000\) units. Neither effect is a single fixed number anymore, both depend on the other channel's spending level.

The Hierarchy Principle

Sometimes an interaction term has a very small p-value while its corresponding main effects do not. The hierarchy principle says that if an interaction is included in a model, the corresponding main effects should be kept as well, even if their own p-values are not significant.

Why this matters: an interaction term, written as a simple product, mathematically already contains the main effects if they are dropped from the model. Removing the main effects does not remove their influence, it just hides it inside a term that is harder to interpret correctly.

Interactions Between Qualitative and Quantitative Predictors

Interactions are not limited to two quantitative predictors. Consider predicting credit card balance from income, quantitative, and student status, qualitative. Without an interaction, the model fits two parallel lines, one for students and one for non-students, with different intercepts but the same slope on income.

Adding an interaction term, income multiplied by the student dummy, allows the two groups to have different slopes as well as different intercepts,

\[ \text{balance}_i \approx (\beta_0 + \beta_2) + (\beta_1 + \beta_3)\times \text{income}_i \quad \text{if student}, \qquad \beta_0 + \beta_1 \times \text{income}_i \quad \text{otherwise} \]

In the credit data, the fitted slope for students turns out lower than for non-students, increases in income are associated with smaller increases in balance for students than for non-students.

Non-linear Effects, Polynomial Regression

Linear regression assumes a straight line or flat plane relationship, but some relationships are clearly curved. Polynomial regression extends the linear model by adding powers of a predictor as extra terms, while still fitting the coefficients by ordinary least squares.

\[ \text{mpg} = \beta_0 + \beta_1 \times \text{horsepower} + \beta_2 \times \text{horsepower}^2 + \varepsilon \]
CoefficientStd. Errort-statisticp-value
Intercept56.90011.800431.6< 0.0001
horsepower-0.46620.0311-15.0< 0.0001
horsepower\(^2\)0.00120.000110.1< 0.0001

This is still technically "linear" regression, since it is linear in the coefficients \(\beta_0, \beta_1, \beta_2\), even though the fitted curve in terms of \(x\) is no longer a straight line.

Polynomial Regression Connects Back to Module 2

This is exactly the polynomial curve fitting idea from Module 2, \(y(x,w) = \sum_{m=0}^{M} w_m x^m\), now applied as a practical extension of linear regression rather than as an abstract illustration of overfitting.

The same warning still applies: adding higher and higher powers of \(x\) will always reduce training RSS, but, exactly as in Module 2's M = 9 polynomial example, a high degree polynomial can fit the training data closely while generalizing poorly. Degree should be chosen using held out data, not by chasing the lowest training RSS.

Model Diagnostics, What Can Go Wrong

A fitted regression can look reasonable on paper, a significant F-statistic, a decent \(R^2\), and still be misleading. Four classic problems are worth checking for routinely.

Non-linearity

  • The true relationship is curved, not straight
  • Detected by plotting residuals against fitted values, a clear curved pattern signals trouble

Non-constant error variance

  • Residual spread fans out as predicted values grow
  • Standard errors and hypothesis tests become unreliable

Outliers and High Leverage Points

An outlier is a point whose response value \(y_i\) is far from what the model predicts, an unusual point in the \(y\) direction. A single severe outlier can distort RSE and \(R^2\), even if it barely changes the fitted line itself.

A high leverage point is different, it has an unusual predictor value \(x_i\), far from the bulk of the other observations. High leverage points can pull the fitted line strongly toward themselves, sometimes changing the slope dramatically even though their residual looks small.

A locally relatable example: in a dataset of microfinance loan sizes and default rates drawn mostly from branches with loan sizes under 50,000 taka, one branch with an unusually large average loan size of 5,000,000 taka would be a high leverage point, regardless of whether its default rate looks unusual.

Collinearity

Collinearity occurs when two or more predictors are closely related to each other. It does not bias the coefficient estimates, but it inflates their standard errors, sometimes dramatically, making individual coefficients look statistically insignificant even when the predictors, together, clearly matter.

Why this is dangerous in practice: with strong collinearity, a small change in the data can swing a coefficient's estimated value wildly, and which of two collinear predictors gets the "credit" in the fitted model can become almost arbitrary.

This is the same underlying issue noted earlier with newspaper and radio spend in the advertising data, two correlated predictors made it hard to tell which one was truly driving sales.

What Linear Regression Cannot Do Well

  • It struggles when the true relationship is strongly non-linear and no polynomial or transformation captures it
  • Outliers and high leverage points can distort it without warning, unless checked for explicitly
  • Collinear predictors make individual coefficients hard to trust
  • It assumes a fixed, additive or interaction structure decided in advance, it does not learn that structure automatically
Looking ahead: later modules introduce methods built to address these exact limitations, logistic regression and support vector machines for classification, kernel smoothing and splines for flexible non-linearity, tree based methods for automatically capturing interactions, and ridge regression and the lasso for regularized fitting when predictors are numerous or collinear.
